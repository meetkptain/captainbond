## 3. MODE PRÉSENTIEL

### 🎯 Concept
> *"Un téléphone qui circule de main en main, portant à chaque fois une question qui rapproche."*

### 📱 Solution Technique
- **1 seul téléphone** (celui de l'hôte)
- **Rotation entre joueurs** (bâton de parole)
- **Inscription rapide des noms** (30 sec)
- **Pas de compte, pas de profil** (juste les prénoms)

### 🎮 Les 3 Modes de Flux

#### **Mode Manuel**
- L'hôte appuie sur "Suivant" quand il veut
- Pour : groupes qui veulent contrôler le rythme

#### **Mode Automatique (RECOMMANDÉ)**
- Timer + transition auto après 5 sec
- Pour : fluidité maximale
- Sablier poétique (pas de chiffres stressants)

#### **Mode Pilote**
- Questions s'enchaînent toutes les X minutes
- Pour : digital detox total

### ⏳ Le Sablier Intelligent

**Caractéristiques :**
- Durée par défaut : 2 min (adaptée par mode)
- Transition auto après 5 sec (mode auto)
- Bouton Pause toujours visible
- Bouton Suivant toujours visible
- Compteur de joueurs qui ont répondu

**Durées par mode :**
| Mode | Durée |
|---|---|
| Icebreaker | 1 min |
| Spicy | 1 min 30 |
| Deep Connection | 3 min |
| Imposteur | 1 min |
| Family | 1 min 30 |
| Date Night | 2 min |

**Design :** Sablier SVG animé (or #D4AF37) qui se vide progressivement. Pas de chiffres stressants.

## 4. MODE DISTANCIEL

### 🎯 Concept
> *"Un arbre neural qui pousse, un DJ IA qui écoute, un fil qui vous relie."*

### 📱 Solution Technique
- **Multi-téléphones** (chacun son écran)
- **Synchronisation temps réel** (WebSockets)
- **Notifications push** (rituels quotidiens)
- **IA vectorielle** (personnalisation, similarités)
- **DJ IA** (facilitateur intelligent)
- **Arbre neural navigable** (graphe de connexions sémantiques)

### 🌳 L'Arbre Neural (Cœur Premium)

**Concept :** Chaque partenaire a un arbre qui pousse avec les conversations. Mais ce n'est pas un arbre botanique hiérarchique. C'est un **réseau de neurones sémantiques** où chaque réponse est un nœud connecté aux autres par similarité.

**Vision :**
> *"Les relations ne sont pas des arbres généalogiques. Elles sont des forêts de connexions."*

**Structure :**
- Chaque réponse = un **nœud**
- Chaque similarité > 0.8 = une **connexion**
- L'ensemble forme un **graphe** qui ressemble à un arbre vu de dessus
- Pas de hiérarchie, juste des liens sémantiques dans tous les sens

**Les 5 Types de Nœuds (pas des couches) :**

| Type | Couleur | Représente | Connexions possibles |
|---|---|---|---|
| 🌱 **Racine** | Brun | Passé, enfance, fondations | Se connecte à plusieurs branches, feuilles, fruits |
| 🪵 **Tronc** | Vert foncé | Valeurs, caractère | Se connecte à fruits, feuilles, racines |
| 🌿 **Branche** | Vert clair | Rêves, projets | Se connecte à racines (origine), fruits, feuilles |
| 🍃 **Feuille** | Vert tendre | Humeur du jour | Se connecte à émotions similaires, racines, troncs |
| 🍎 **Fruit** | Or/Rouge | Moments de résonance | Intersection de plusieurs nœuds (similarité > 0.9) |

**La différence clé :** Un nœud "feuille" peut se connecter directement à un nœud "racine" si le thème est similaire. **Pas de hiérarchie, juste des liens sémantiques.**

### 📅 Le Fil du Jour (Rituel Quotidien)

**Structure hebdomadaire :**
- **Lundi → Vendredi** : 1 question à 20h (générée par le DJ IA)
- **Samedi** : Pas de question (vivre sans l'app)
- **Dimanche** : 3-5 questions profondes (rituel hebdo, séquencé par le DJ IA)

**Règles d'or :**
- Pas de notif agressive (juste changement d'icône)
- Pas de streak (pas de culpabilisation)
- Pas de rattrapage (on ne revient pas en arrière)
- Silence respecté (pas de relance)

### 🎯 Mode Profiling (Introspection)

**Concept :** Chacun répond chez soi, l'IA analyse.

**Fonctionnalités :**
- Questions de profiling (valeurs, peurs, rêves)
- IA vectorielle analyse les réponses
- Miroir sémantique : "Vous partagez 3 valeurs communes"
- Suggestions de questions pour approfondir

**Usage :** Préparation au lien (avant de rejoindre un groupe)

### 👥 Groupe à Distance

**Concept :** Amis/famille éloignés géographiquement.

**Fonctionnalités :**
- Synchronisation N téléphones
- Modes adaptés (Imposteur, Icebreaker)
- Chat vocal intégré (optionnel)
- Replay des sessions

## 5. L'ARBRE NEURAL NAVIGABLE

### 🎯 Concept
> *"L'arbre n'est pas une image. C'est une carte de votre relation. Et comme toute carte, il faut pouvoir la lire, l'explorer, la comprendre."*

### 🧭 Les 5 Niveaux de Navigation

#### **Niveau 1 : Vue Globale (La Forêt)**
- L'arbre complet en vue d'ensemble
- Zoom in/out, pan, rotation
- Vue "satellite"

#### **Niveau 2 : Exploration de Nœud (Le Détail)**
- Clic sur un nœud → voir la réponse complète
- Contexte (date, émotion, thème)
- Connexions directes

#### **Niveau 3 : Filtrage par Type (La Carte Thématique)**
- Filtrer par type de nœud (racines, troncs, branches, feuilles, fruits)
- Filtrer par émotion (joie, peur, tristesse, etc.)
- Filtrer par thème (famille, travail, amour, etc.)
- Filtrer par période (ce mois, cette année, etc.)

#### **Niveau 4 : Parcours Guidé (Le Storytelling)**
- L'arbre se "raconte" automatiquement
- Zoom progressif sur les zones importantes
- Types : chronologique, thématique, émotionnel, connexions

#### **Niveau 5 : Recherche & Découverte (L'Explorateur)**
- Barre de recherche (mot-clé, émotion, date)
- Suggestions automatiques ("Vous n'avez pas exploré le thème 'travail' depuis 2 mois")
- Zones non explorées mises en évidence

### 🎨 Les Outils de Compréhension

#### **La Légende Interactive**
- Toujours visible en bas de l'écran
- Types de nœuds + émotions

#### **Les Tooltips Contextuels**
- Au survol d'un nœud : résumé + émotion + connexions

## 6. LE DJ IA DANS L'ARBRE

### 🎯 Concept
> *"L'IA n'est pas un quiz master. C'est un DJ qui mixe les questions en fonction de l'ambiance, de l'histoire, et de l'énergie du moment."*

### 🧠 Le Rôle du DJ IA

Le DJ IA ne se contente pas de poser des questions aléatoires. Il :

1. **Analyse l'arbre neural** (nœuds, connexions, clusters)
2. **Détecte les patterns** (thèmes récurrents, émotions dominantes)
3. **Génère du contenu sur mesure** pour le couple
4. **Adapte le rythme** (pas trop profond trop vite)
5. **Crée des ponts** entre vos réponses passées et présentes

### 🎨 Les 5 Niveaux d'Intelligence

#### **Niveau 1 : Questions Quotidiennes Contextuelles**

**Au lieu de :** "Comment s'est passée ta journée ?" (générique)

**Le DJ IA propose :**
```
Contexte détecté :
- Hier : Alice a répondu "J'ai stressé au boulot"
- Bob a répondu "J'ai pensé à toi"
- Arbre : cluster "soutien" en croissance

Question générée :
"Qu'est-ce qui t'a fait penser à moi hier ?"
```

**Pourquoi c'est intelligent :**
- ✅ Se souvient des réponses d'hier
- ✅ Crée un lien entre vos journées
- ✅ Explore un cluster de l'arbre (soutien)

#### **Niveau 2 : Exploration des Clusters de l'Arbre**

**Le DJ IA détecte les zones denses de l'arbre et les explore.**

**Exemple :**
```
Arbre analysis :
- Cluster "peur" : 8 nœuds, très dense
- Cluster "joie" : 3 nœuds, en croissance
- Cluster "projet" : 2 nœuds, récent

DJ IA décide :
→ Explorer le cluster "peur" (le plus dense)
→ Mais doucement (pas trop intense)

Question générée :
"Tu as souvent parlé de peur ces dernières semaines. 
Qu'est-ce qui t'a aidé à la traverser ?"
```

**Pourquoi c'est intelligent :**
- ✅ Utilise la structure de l'arbre pour guider l'exploration
- ✅ Respecte le rythme (pas de forcing)
- ✅ Approfondit ce qui est important pour vous

#### **Niveau 3 : Suivi des Fruits de Résonance**

**Quand un fruit apparaît (similarité > 0.9), le DJ IA le célèbre et l'explore.**

**Exemple :**
```
Fruit détecté :
- Alice : "Je me sens aimé quand tu me fais rire"
- Bob : "Je me sens aimé quand tu me fais rire"
- Similarité : 0.96
- Fruit d'or créé dans l'arbre

DJ IA génère (le lendemain) :
"Hier, vous avez tous les deux parlé de rire. 
Quel est le fou rire le plus mémorable que vous avez partagé ?"
```

**Pourquoi c'est intelligent :**
- ✅ Célèbre les moments de connexion
- ✅ Approfondit ce qui vous relie
- ✅ Crée une narration continue

#### **Niveau 4 : Contenu Généré à Partir de Vos Réponses**

**Le DJ IA utilise vos propres mots pour créer de nouvelles questions.**

**Exemple :**
```
Réponses récentes :
- Alice : "J'ai besoin de plus de calme"
- Bob : "Je me sens dispersé"

DJ IA détecte : thème "calme vs dispersion"

Question générée :
"Vous avez tous les deux parlé de calme et de dispersion. 
Qu'est-ce qui vous aide à trouver votre centre ?"
```

**Pourquoi c'est intelligent :**
- ✅ Utilise VOS mots (pas des questions pré-faites)
- ✅ Crée des ponts entre vos réponses
- ✅ Personnalisation extrême

#### **Niveau 5 : Adaptation au Rythme de l'Arbre**

**Le DJ IA respecte la croissance de l'arbre.**

**Règles :**
```
Si l'arbre a < 20 nœuds :
→ Questions légères (icebreaker, découverte)
→ Pas trop profond trop vite

Si l'arbre a 20-100 nœuds :
→ Mixte (léger + profond)
→ Exploration des premiers clusters

Si l'arbre a > 100 nœuds :
→ Questions profondes autorisées
→ Exploration des clusters denses
→ Vulnérabilité acceptée
```

**Exemple :**
```
Couple débutant (15 nœuds) :
DJ IA : "Quel est ton souvenir d'enfance préféré ?" (léger)

Couple expérimenté (150 nœuds) :
DJ IA : "Quelle peur te limite encore aujourd'hui ?" (profond)
```

**Pourquoi c'est intelligent :**
- ✅ Respecte la maturité de la relation
- ✅ Ne brusque pas les nouveaux couples
- ✅ Challenge les couples installés

### 📅 Exemple sur 7 Jours

#### **Jour 1 : Lundi (Rituel Quotidien)**

**Contexte :**
- Arbre : 45 nœuds
- Cluster dominant : "travail" (8 nœuds)
- Émotion dominante : "stress"

**Question générée :**
> *"Qu'est-ce qui t'a le plus stressé au travail cette semaine ?"*

**Pourquoi :**
- ✅ Explore le cluster dominant
- ✅ Adapte à l'émotion dominante
- ✅ Timing parfait (lundi, début de semaine)

#### **Jour 2 : Mardi**

**Question générée :**

**Pourquoi :**

#### **Jour 3 : Mercredi**

**Contexte :**

**Question générée :**

async function generateDailyQuestion(coupleId: string) {
  // 1. Récupérer l'arbre du couple
  const tree = await getTree(coupleId);
  
  // 2. Analyser l'arbre
  const analysis = {
    nodeCount: tree.nodes.length,
    clusters: await detectClusters(tree),
    recentThemes: await getRecentThemes(tree, 7),
    recurringThemes: await getRecurringThemes(tree, 30),
    resonanceMoments: await getResonanceMoments(tree),
    dominantEmotion: await getDominantEmotion(tree, 7),
    coupleProfile: await getCoupleProfile(coupleId),
  };
  
  // 3. Décider du type de question
  let questionType;
  
  if (analysis.nodeCount < 20) {
    questionType = 'discovery';
  } else if (analysis.resonanceMoments.length > 0) {
    questionType = 'resonance';
  } else if (analysis.recurringThemes.length > 0) {
    questionType = 'deep';
  } else {
    questionType = 'contextual';
  }
  
  // 4. Générer la question
  const question = await generateQuestion(questionType, analysis);
  
  return question;
}

async function generateQuestion(type: string, analysis: any) {
  const prompt = `
    Tu es le DJ IA de Captain Bond, un facilitateur de connexion bienveillant.
    
    Contexte du couple :
    - Nombre de nœuds dans l'arbre : ${analysis.nodeCount}
    - Clusters détectés : ${analysis.clusters.map(c => c.theme).join(', ')}
    - Thèmes récents : ${analysis.recentThemes.join(', ')}
    - Thèmes récurrents : ${analysis.recurringThemes.join(', ')}
    - Émotion dominante : ${analysis.dominantEmotion}
    - Style de communication : ${analysis.coupleProfile.style}
    
    Type de question à générer : ${type}
    
    Règles :
    - Si type = 'discovery' : question légère, découverte mutuelle
    - Si type = 'resonance' : explorer un moment de résonance récent
    - Si type = 'deep' : explorer un thème récurrent en profondeur
    - Si type = 'contextual' : adapter à l'émotion dominante
    
    Génère UNE question en français, max 20 mots, qui :
    - Est personnalisée au couple
    - Respecte leur rythme (pas trop profond si débutant)
    - Encourage la vulnérabilité sans forcer
    - Utilise leurs propres mots si possible
  `;
  
  const response = await openai.chat.completions.create({
    model: 'gpt-4o',
    messages: [{ role: 'user', content: prompt }],
    temperature: 0.7,
  });
  
  return response.choices[0].message.content;
}

### 🗄️ Schéma de Base de Données

```sql
-- Utilisateurs
CREATE TABLE users (
  id UUID PRIMARY KEY,
  email VARCHAR(255) UNIQUE,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Couples (pour mode distanciel)
CREATE TABLE couples (
  id UUID PRIMARY KEY,
  partner_a_id UUID REFERENCES users(id),
  partner_b_id UUID REFERENCES users(id),
  communication_style VARCHAR(50), -- détecté par DJ IA
  maturity_level INT, -- basé sur nombre de nœuds
  created_at TIMESTAMP DEFAULT NOW()
);

-- Arbres neuraux
CREATE TABLE trees (
  id UUID PRIMARY KEY,
  couple_id UUID REFERENCES couples(id),
  user_id UUID REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Nœuds de l'arbre (réseau neural)
CREATE TABLE tree_nodes (
  id UUID PRIMARY KEY,
  tree_id UUID REFERENCES trees(id),
  response_id UUID REFERENCES responses(id),
  node_type VARCHAR(20), -- 'root', 'trunk', 'branch', 'leaf', 'fruit'
  theme VARCHAR(100),
  emotion VARCHAR(50),
  embedding VECTOR(1536),
  position_x FLOAT,
  position_y FLOAT,
  size FLOAT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Connexions entre nœuds
CREATE TABLE tree_connections (
  id UUID PRIMARY KEY,
  node_a_id UUID REFERENCES tree_nodes(id),
  node_b_id UUID REFERENCES tree_nodes(id),
  similarity FLOAT, -- similarité cosinus (0.8 - 1.0)
  connection_type VARCHAR(20), -- 'semantic', 'temporal', 'emotional'
  created_at TIMESTAMP DEFAULT NOW()
);

-- Signets (marque-pages dans l'arbre)
CREATE TABLE tree_bookmarks (
  id UUID PRIMARY KEY,
  tree_id UUID REFERENCES trees(id),
  user_id UUID REFERENCES users(id),
  node_id UUID REFERENCES tree_nodes(id),
  title VARCHAR(255),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Profil DJ IA du couple
CREATE TABLE dj_profiles (
  id UUID PRIMARY KEY,
  couple_id UUID REFERENCES couples(id),
  dominant_emotions JSONB, -- émotions récurrentes
  recurring_themes JSONB, -- thèmes récurrents
  communication_style VARCHAR(50),
  depth_preference VARCHAR(20), -- 'light', 'medium', 'deep'
  last_question_type VARCHAR(20),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Historique des questions du DJ IA
CREATE TABLE dj_questions (
  id UUID PRIMARY KEY,
  couple_id UUID REFERENCES couples(id),
  question_type VARCHAR(20), -- 'discovery', 'resonance', 'deep', 'contextual'
  question_text TEXT,
  generated_by VARCHAR(20), -- 'ai', 'therapist', 'library'
  context_snapshot JSONB, -- contexte au moment de la génération
  created_at TIMESTAMP DEFAULT NOW()
);

-- Questions
CREATE TABLE questions (
  id UUID PRIMARY KEY,
  mode VARCHAR(50), -- 'icebreaker', 'deep', 'impostor', etc.
  context VARCHAR(50), -- 'presentiel', 'distanciel'
  text TEXT,
  intensity INT, -- 1-3
  duration INT, -- en secondes
  created_by UUID REFERENCES users(id) -- thérapeute si signé
);

-- Réponses
CREATE TABLE responses (
  id UUID PRIMARY KEY,
  question_id UUID REFERENCES questions(id),
  user_id UUID REFERENCES users(id),
  couple_id UUID REFERENCES couples(id),
  text TEXT,
  embedding VECTOR(1536), -- pour IA vectorielle
  created_at TIMESTAMP DEFAULT NOW()
);

-- Abonnements
CREATE TABLE subscriptions (
  id UUID PRIMARY KEY,
  couple_id UUID REFERENCES couples(id),
  payer_user_id UUID REFERENCES users(id),
  status VARCHAR(20), -- 'active', 'paused', 'cancelled'
  plan VARCHAR(20), -- 'monthly', 'yearly'
  current_period_start TIMESTAMP,
  current_period_end TIMESTAMP
);

-- Sessions de jeu (mode présentiel)
CREATE TABLE game_sessions (
  id UUID PRIMARY KEY,
  mode VARCHAR(50),
  players JSONB, -- liste des prénoms
  flow_mode VARCHAR(20), -- 'manual', 'automatic', 'pilot'
  started_at TIMESTAMP,
  ended_at TIMESTAMP
);
```
