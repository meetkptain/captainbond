# Blog Process — full operations guide (create / enrich / SEO-GEO / audit)

This is the single reference for operating the Captain Bond blog. It covers the
content-layer architecture, the daily 3-command loop, legacy migration, audit
expectations, and guardrails. Quick start lives in `docs/BLOG_WORKFLOW.md`.

---

## 1. Architecture (why it is easy + robust)

- **All content is pure data.** Every article is a typed `BlogPost` object in
  `src/content/blog/*.ts`. Authors (agents or humans) **never write JSX**.
- **One renderer.** `src/components/blog/BlogArticle.tsx` derives ALL structure
  and SEO/GEO from the data fields: FAQPage JSON-LD, `SpeakableSpecification`
  (`cssSelector .article-body h2`), the "Why it matters for AI search" geoBlock
  section, TOC, takeaways, author bio, CTA.
- **SSG.** `src/app/(marketing)/blog/[slug]/page.tsx` + `src/app/fr/blog/[slug]/page.tsx`
  (`runtime='nodejs'`, `dynamic='force-static'`, `dynamicParams=false`) prerender
  every article to static HTML at build time.
- **Index is generated.** `scripts/sync-blog.ts` regenerates `src/content/blog/index.ts`
  (one `import` per file, keyed by filename — not slug — so EN/FR pairs with the
  same slug both resolve, e.g. `questions-pour-couple.ts` + `questions-pour-couple-en.ts`).
- **Single source for SEO meta + sitemap.** `[slug]` `generateMetadata` builds
  `hreflang` (x-default/en/fr) from `frSlug`. `src/app/sitemap.ts` emits 102
  alternates with `lastMod = modified ?? published`.

Result: adding/editing an article = edit a `.ts` data file + run scripts. No
component, route, or schema changes.

## 2. The 3-command loop (daily work)

```bash
# 1) CREATE a paired EN/FR article (scaffold with TODOs)
npm run blog:new -- --en "How to Host a Game Night" \
                     --fr "Comment organiser une soirée jeux" \
                     --slug how-to-host-game-night \
                     --frslug organiser-soiree-jeux \
                     --hub party          # party | couple | bar

# 2) ENRICH SEO + GEO (generic, idempotent, hub/localized templates)
#    Auto-fills: related, geoBlock, faq (3 Q&A >=40c), endingQuestion.
#    Reads files from DISK, so a brand-new article enriches without sync first.
#    Never overwrites real copy.
npm run blog:enrich

# 3) BUILD + AUDIT (regenerate index, generate OG images, run audit)
npm run blog:build
```

Goal after step 3: `Audit: 34 content + 0 legacy = 34 posts | 0 errors | 0 warnings`
and `GEO: 34/34 content posts have a geoBlock (0 missing).`

## 4. What an agent fills vs what is automated

**Automated by `blog:enrich`** (safe, template-based, localized by hub×locale):
`related`, `geoBlock`, `faq` (when empty / all-TODO), `endingQuestion`.

**Must be written by a human/agent** (real copy — audit flags these as TODO):
`description`, `sections[].p`, `takeaways`.

**Never touch:** `slug`, `frSlug`, `ogImage`, `published`. Dates drive `lastmod`
in the sitemap; migration preserves legacy dates and must not reset them.

---

## 5. Article anatomy (fields of `BlogPost`)

See `src/lib/content/types.ts`. Key fields:
- `slug` / `locale` / `frSlug` — routing + hreflang counterpart.
- `title` / `description` — meta title + 150–160 char meta description.
- `ogImage` — `/og/blog-<slug>-<loc>.webp` (generated by `blog:build`).
- `published` / `modified` — ISO dates; `modified ?? published` → sitemap `lastmod`.
- `hub` — `party | couple | bar`; drives internal linking, OG style, `articleSection`.
- `faq[]` — FAQPage JSON-LD; answers ≥40 chars = AI-citation fuel.
- `takeaways[]` — shown as a callout after the header.
- `sections[]` — `{ h2, p?, list?, quote? }`; `h2` becomes Speakable selector + TOC anchor.
- `related[]` — internal links (auto-filled by hub/locale).
- `cta` — closing call-to-action.
- `geoBlock` — GEO / AI-citation paragraph (auto-filled by template).
- `endingQuestion` — localized closing question (auto-filled by template).

## 6. Legacy migration (reference — already done for all 26)

`scripts/migrate-one.ts` converts a legacy `page.tsx` into a content-layer `.ts`:
- Preserves `published`/`modified` from `meta.other?.datePublished/.dateModified`
  (never resets to today).
- Extracts the real OG path from `meta.openGraph.images[0].url` (not a slug guess).
- Captures `<section>` prose (`<p>` + `<li>`), not just `<h2>`.
- Injects external string-array consts into the `faqSchema` eval scope (fixes
  `ReferenceError: <array> is not defined`).
- On EN/FR slug collision, suffixes the EN file `-en` (e.g. `questions-pour-couple-en.ts`).

Run per pair: `npm run blog:migrate` (edit the source path inside the script or
extend it to loop). After migrating, run `blog:enrich` + `blog:build`.

## 7. Audit expectations & guardrails

`npm run blog:audit` (also run inside `blog:build`) enforces the following on
every post. **err** blocks the build/merge; **warn** is non-blocking but must be
cleaned before shipping.

| Field | Rule | Level |
|---|---|---|
| `title` | ≥10 chars | err |
| `description` | ≥50 chars | warn |
| `ogImage` | file exists in `public/` | err |
| `faq` | ≥3 Q&A, each `q`+`a` present | err |
| `faq[].a` | non-TODO, ≥40 chars (AI-citation quality) | warn |
| `sections` | ≥3 blocks with real `p` | err |
| `takeaways` | ≥3, none `TODO` / <5 chars | warn |
| `related` | ≥1 internal link, routes valid | warn |
| `frSlug` | has a counterpart post (EN↔FR) | err |
| `geoBlock` | non-TODO, ≥40 chars | warn |
| `endingQuestion` | present (filled by `blog:enrich`) | (via enrich) |

- **SEO:** `faq` answers ≥40 chars, `description` ≥50, `sections` with real
  `p`, `takeaways` filled.
- **GEO:** `geoBlock` ≥40 chars — **TODO-aware** (`geoBlock: 'TODO…'` = gap), so
  nothing ships silently incomplete.
- **Locales:** every EN post needs a `frSlug` counterpart and vice-versa; the
  audit **errors** if an `frSlug` has no matching post.
- **CI:** `.github/workflows/blog-audit.yml` runs `blog:build` on push — a red
  audit blocks merge.

## 8. Critical conventions (robustness)

- **Never overwrite `published`/`modified`** with today's date on migration.
- **`blog:enrich` is idempotent** — running it twice is safe; it only fills
  TODO/empty/short fields.
- **Dates:** `lastMod = modified ?? published`; legacy dates are real 2025 dates.
- **hreflang:** always 3 entries (x-default → EN, en → EN, fr → `/fr/...`).
- **Add a new hub:** extend the `GEO` / `FAQ` / `ENDING` maps in
  `scripts/blog-enrich.ts` (one block per hub × locale). No other change needed.
- **OG images:** generated by `blog:build` from the `hub` style; do not hand-edit
  `public/og/blog-*.webp`.

## 9. Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `blog:enrich` fills nothing on a new article | Old version read `allPosts` (generated index) — new file not synced yet | Use current disk-based `blog:enrich` (reads files directly) |
| `Top-level await not supported (cjs)` | `blog-enrich.ts` used top-level `await` | Wrapped in `async function main()` + `main()` |
| Duplicate import / EN post dropped from `allPosts` | `sync-blog` keyed import by slug, not filename | Import path + var name use filename (`toIdent(p.file)`) |
| `ReferenceError: X is not defined` in extractFaq | faqSchema maps external string arrays | `migrate-one` injects all top-level string-array consts into eval scope |
| EN overwritten by FR on same slug | Same filename for both locales | Suffix EN file `-en` in `migrate-one` |
| Audit reports 34/34 but geoBlock is `TODO` | Audit not TODO-aware | `audit-blog` flags `/TODO/i` in geoBlock |
