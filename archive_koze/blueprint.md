## 1. LA VISION

### Ce qu'est KOZÉ

**KOZÉ** est un jeu de soirée numérique où chaque joueur utilise son téléphone comme manette. L'écran principal (TV, ordinateur) affiche le jeu, et les téléphones deviennent des contrôleurs individuels.

> *"KOZÉ n'est pas un jeu. C'est un prétexte pour se retrouver, rire ensemble, et célébrer ce qui nous relie."*

### La Promesse

> *"Transformez n'importe quelle soirée en un moment mémorable où les gens rient, débattent, se chambrent, et repartent avec des souvenirs communs."*

### Les Valeurs

| Valeur | Ce que ça signifie |
| :--- | :--- |
| **Authenticité** | 100% ancré dans la culture réunionnaise |
| **Simplicité** | On joue en 10 secondes, sans inscription |
| **Rassemblement** | Le jeu crée du lien, pas de la compétition |
| **Bienveillance** | Tout le monde peut jouer, personne n'est exclu |
| **Fierté** | Célébrer la culture locale avec humour et respect |

## 2. LE CONCEPT

### Comment ça marche (en 4 étapes)

| Étape | Ce qui se passe |
| :--- | :--- |
| **1. L'hôte crée une salle** | Sur son ordinateur, il va sur le site de KOZÉ. Il clique sur "Créer une salle". Un code à 4 lettres est généré (ex: "3K9A"). |
| **2. Les joueurs rejoignent** | Chaque invité sort son téléphone, va sur le même site, tape le code "3K9A" et entre son prénom. Tout le monde est dans la partie en 10 secondes. |
| **3. On joue** | L'hôte choisit un mode. Les questions ou défis s'affichent sur l'écran TV. Chaque joueur répond sur son téléphone. Les réponses sont révélées en direct. |
| **4. On recommence** | Une partie dure 15-20 minutes. À la fin, un gagnant est désigné. Mais surtout, on a passé un bon moment ensemble. |

### Pourquoi ça marche

| Raison | Explication |
| :--- | :--- |
| **Zéro installation** | Les joueurs n'ont rien à télécharger. Juste un navigateur web. |
| **Zéro inscription** | Pas de compte, pas d'email, pas de mot de passe. Juste un prénom. |
| **Tout le monde joue** | 2 à 8 joueurs. Les enfants, les grands-parents, les amis... tout le monde peut participer. |
| **Le téléphone comme bouclier** | Les timides ne sont pas exposés. Les réponses sont anonymes jusqu'à la révélation. |
| **Le rythme est soutenu** | Pas de temps mort. Les manches s'enchaînent rapidement. |

## 3. LES MODES DE JEU

### Modes Gratuits (MVP - 3 modes)

#### 1. "Vrai ou Fau" - Le Social

| Aspect | Détail |
| :--- | :--- |
| **Principe** | Une affirmation s'affiche. Chaque joueur vote Vrai ou Faux. La réponse est révélée. Ceux qui ont juste marquent des points. |
| **Durée** | 30 secondes par manche |
| **Émotion** | La tension du vote, les cris de la révélation |
| **Pourquoi c'est génial** | C'est le plus simple. Tout le monde comprend en 10 secondes. |

**Exemple :**
> *"Le Piton de la Fournaise est entré en éruption 12 fois en 2025."*
> → Réponse : **FAUX** (seulement 7 fois)

---

#### 2. "Le Grand Débat" - Le Politique

| Aspect | Détail |
| :--- | :--- |
| **Principe** | Une question clivante est posée. Chaque joueur tape son argument. Les arguments sont affichés (anonymes). On vote pour le meilleur. |
| **Durée** | 50 secondes par manche |
| **Émotion** | La passion des débats, les fous rires des arguments absurdes |
| **Pourquoi c'est génial** | Les gens ADORENT donner leur avis. Le jeu devient un prétexte à discuter. |

**Exemple :**
> *"Le meilleur rougail, c'est au saucisse ou au poisson ?"*
> Argument gagnant : *"Le rougail saucisse, c'est le plat du dimanche chez mamie. Ça sent la maison, ça sent l'enfance. Point final."*

---

#### 3. "Quiz Flash" - Le Rapide

| Aspect | Détail |
| :--- | :--- |
| **Principe** | Une question à choix multiple est posée. Le plus rapide avec la bonne réponse gagne. |
| **Durée** | 20 secondes par manche |
| **Émotion** | L'urgence, la compétition amicale |
| **Pourquoi c'est génial** | Simple, rapide, tout le monde peut jouer. |

**Exemple :**
> *"Quelle est la capitale de La Réunion ?"*
> A. Saint-Pierre | B. Saint-Denis | C. Saint-Paul | D. Saint-Louis
> → Réponse : **B. Saint-Denis**

---

### Modes Premium (Pack Péi - 5 modes supplémentaires)

| Mode | Principe | Pourquoi c'est génial |
| :--- | :--- | :--- |
| **L'Imposteur Kréol** | Un mot est affiché. Un joueur est l'imposteur (il ne voit pas le mot). Il doit poser des questions pour deviner sans se faire repérer. | La tension est maximale. Les accusations fusent. |
| **Le Bac à Sable** | Une lettre est tirée. Chaque joueur doit trouver un mot dans chaque catégorie (prénom, animal, objet, expression, plat, lieu). | Un classique de l'enfance. La version numérique ajoute le chronomètre. |
| **Le Cadavre Exquis** | Une phrase est commencée. Chaque joueur ajoute 3 mots. Le résultat est une histoire absurde. | Collaboratif, pas compétitif. Le résultat est unique à chaque partie. |
| **Le Défi Photo** | Une photo s'affiche. Les joueurs doivent reconnaître le lieu, le plat ou l'animal le plus vite possible. | Visuel, accessible à tous, même les enfants. |
| **Le Gage Party** | Le perdant d'une manche tire un défi sur une roue. Il doit le réaliser (ou passer sans conséquence). | Crée des souvenirs dont on se souvient des années après. |

### Modes Premium (Pack Soirée - 3 modes supplémentaires)

| Mode | Principe | Pourquoi c'est génial |
| :--- | :--- | :--- |
| **Le Mime** | Un mot est affiché. Un joueur doit le mimer. Les autres devinent. | Tout le monde devient acteur. Les fous rires sont garantis. |
| **L'Enquête** | Un "crime" a été commis. Un joueur est le coupable. Les autres doivent l'interroger pour le trouver. | Le jeu de rôle est toujours prenant. |
| **Le Story Teller** | 3 images sont affichées. Chaque joueur invente une histoire qui les relie. | Les gens adorent raconter des histoires. |

## 4. L'EXPÉRIENCE UTILISATEUR

### L'Écran TV (Ce que tout le monde regarde)

| Élément | Description |
| :--- | :--- |
| **Le code de la salle** | Affiché en grand pour que les retardataires rejoignent facilement |
| **La question ou le défi** | En gros caractères, lisible à distance |
| **Le timer** | Une barre de progression qui crée la tension |
| **Les votes en direct** | Affichés en temps réel (anonymes) |
| **La révélation** | Animations de confettis, de fumée volcanique |
| **Le classement** | Les scores de tous les joueurs, mis à jour en direct |
| **Le chat** | Les commentaires et les chambrages en direct |

### Le Téléphone (Ce que chaque joueur voit)

| Élément | Description |
| :--- | :--- |
| **La question** | Le texte de la question |
| **Le timer** | Un compte à rebours pour répondre |
| **Les boutons d'action** | "Vrai/Faux", les choix A/B/C/D, un champ texte |
| **Le score personnel** | Ses propres points en haut de l'écran |
| **Le chat** | Pour envoyer des messages aux autres |

### Le Flow d'une Soirée Type

```
1. Installation (1 minute)
   - L'hôte branche son ordinateur sur la TV
   - Tout le monde sort son téléphone

2. Création de la salle (30 secondes)
   - L'hôte crée une salle sur son ordinateur
   - Un code à 4 lettres apparaît

3. Rejoindre la salle (1 minute)
   - Chaque joueur va sur le site, tape le code, entre son prénom

4. Choix du mode (30 secondes)
   - L'hôte choisit un mode de jeu (ex: "Vrai ou Fau")

5. Manche 1, 2, 3... (15-20 minutes)
   - Les questions s'enchaînent
   - Les scores évoluent
   - Les rires fusent

6. Fin de la partie (2 minutes)
   - Un gagnant est désigné
   - Un récap' des moments forts s'affiche
   - Tout le monde veut refaire une partie
```

## 11. L'ARCHITECTURE TECHNIQUE

Pour supporter la charge en cas de viralité et garantir des coûts d'exploitation minimes (bloqués à ~32$/mois max), KOZÉ a adopté une architecture Edge / Serverless robuste :

*   **Frontend & APIs (Edge)** : Next.js 16+ (App Router) déployé sur **Cloudflare Pages** (V8 runtime `workerd`).
*   **Temps réel & Base de données** : **Supabase** (PostgreSQL) et **Supabase Realtime** (Presence pour la liste des joueurs et CDC pour la TV, optimisé sans messages de confort pour rester sous les 5M de messages).
*   **Stockage** : **Wasabi S3** (Stockage d'images pour le mode Défi Photo avec signature d'upload direct client via `aws4fetch` pour 0$ de frais de bande passante/egress).
*   **Sécurité** : Politiques de Row-Level Security (RLS) SQL liées à l'authentification anonyme de Supabase.
